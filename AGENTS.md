# AGENTS.md — nix-dots

NixOS + home-manager flake for personal machines. No tests, no CI, no formatter output wired in `flake.nix`.

## Hosts

- `thinkpadt14` — laptop, systemd-boot, uses `mango` + `noctalia` shells.
- `eva02` — desktop, Secure Boot via `lanzaboote`, uses `dms` shell.

Both import `hosts/common.nix` and `home/common.nix`. Per-host settings live in `hosts/<host>/` (`configuration.nix`, `home.nix`, `options.nix`, `hardware-configuration.nix`).

Custom option `my.host.isLaptop` (`home/options.nix`) drives laptop-only widgets (e.g., battery in the DMS bar).

## Rebuild / switch

The helper is `nh` (Nix Helper). Its flake is pinned to `~/nix-dots` in `hosts/common.nix`.

```bash
# Switch to the current host's config without updating inputs
nh os switch

# Switch and update flake inputs
nh os switch --update
```

There's also a wrapper script at `scripts/update` that:

1. Fetches `origin/main` and refuses to switch if local is behind.
2. Runs `nh os switch` or `nh os switch --update`.
3. If a rebuild produced store-backed changes (e.g., `flake.lock`), prompts to commit and push with a `nixos: switch <timestamp>` message.

> Prefer `scripts/update` for normal switches; use `nh os switch` directly only when you intentionally want to skip the remote-sync check.

## Validation

- `nix flake check` — type-checks the flake outputs and NixOS configurations.
- `nix flake show` — lists the two `nixosConfigurations`.

No formatter output is declared in `flake.nix`, so `nix fmt` does not work. Format Nix files manually with `nixfmt` (available in PATH):

```bash
find . -name '*.nix' -exec nixfmt {} +
```

## Code layout

| Directory                  | Purpose                                                                                  |
| -------------------------- | ---------------------------------------------------------------------------------------- |
| `flake.nix`                | Entry point; defines `nixosConfigurations` for both hosts.                               |
| `hosts/<host>/`            | Host-specific NixOS + home-manager imports.                                              |
| `hosts/common.nix`         | Shared NixOS config (`nh`, locale, firewall, experimental features).                     |
| `home/common.nix`          | Shared home-manager imports; base packages.                                              |
| `home/applications/<app>/` | Per-application home-manager modules.                                                    |
| `modules/<category>/`      | Reusable NixOS system modules imported by hosts.                                         |
| `scripts/`                 | Custom scripts symlinked to `~/.local/bin` via `home/applications/zsh/zsh.nix`.          |
| `walls/`                   | Wallpapers; `blackhole-smooth-240x67.dur` is used as the `ly` display-manager animation. |

## Live-config vs Nix-managed

Some apps are configured with `mkOutOfStoreSymlink` so their runtime configs are editable live and do not require a rebuild:

- `home/applications/hyprland/config` → `~/.config/hypr`
- `home/applications/qutebrowser/config` → `~/.config/qutebrowser`
- `home/applications/tmux-sessionizer/config` → `~/.config/tmux-sessionizer`

Most other apps (kitty, zsh, tmux, nixvim, git, firefox, etc.) are configured through Nix options and need a rebuild after changes.

Note: `qutebrowser/theme.py` is generated by Stylix and gitignored; the repo keeps the live-editable `config.py`, `autoconfig.yml`, etc.

## Hyprland config is Lua, not `.conf`

The Hyprland config in `home/applications/hyprland/config/` is written in Lua using a custom `hl` global (see `hyprland.lua` and `hyprland/*.lua`). Do not assume standard Hyprland `.conf` syntax here. Host-specific monitor and workspace overrides live in `home/applications/hyprland/overrides/<host>/` and are symlinked in by `home/applications/hyprland/hyprland.nix`.

## External flake inputs

Key inputs: `nixpkgs`, `home-manager`, `nixvim`, `noctalia`, `dms` / `dms-plugin-registry`, `lanzaboote`, `kimi-code`, `stylix`. Run `nix flake lock --update-input <name>` to bump a single input, then `nh os switch` to test it.

## Backup behavior

`home-manager.backupFileExtension = "backup"`. If a home file already exists on activation, home-manager moves it to `<path>.backup` instead of failing.

## Things to avoid

- Do not edit `hardware-configuration.nix`; it is generated by `nixos-generate-config`.
- Do not commit secrets, age-encrypted files, or private keys.
- `result` and `result-*` are gitignored (Nix build symlinks).
- `home/applications/qutebrowser/config/theme.py` is generated by Stylix and gitignored.
